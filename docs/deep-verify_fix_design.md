# deep-verify 수정안 설계 — Fable 적대 리뷰용 (2026-07-03)

> 목적: `deep-verify_SKILL_neutral.md`(원본 절차 규칙)의 결함에 대한 수정안. 이 문서는 **리뷰 대상** — "이 수정이 구멍을 실제로 닫나 + 새 구멍을 만드나"를 적대적으로 검증받기 위함.
> 원본 스킬: `deep-verify_SKILL_neutral.md` (같은 폴더)

## 고치려는 결함 (재확인용)
- **B1**: 비치명·단독·판단형 finding이 §3.2 트리거 목록(치명단독/B·C충돌/고위험합의) 밖인데, §3.3 판단형 행 3개는 전부 토론 전제 → **verdict 경로 없음**.
- **B2**: 치명+근거불명확+미해결에서 행6(🟡 사용자결정) vs 불변식3(🔴 BLOCK) **충돌** + 우선순위 미정의.
- **B3**: 주장/표면 인벤토리 단계 부재 → §3.2 "고위험 합의" 트리거 **무입력(무발동 가능)** + §4 "미검증 영역" 전수성 검증 불가.
- **B4**: 머지 게이트 스킬인데 **최종 통과/불통과 집계 규칙 없음**(finding 단위 판정만).
- **I2**: 3축→등급 매핑·stake 순서 없음 → "고정 함수"가 판단형에서 샘 + 행7 순환.
- **I6**: 근거 필드 "존재"만 검사, "실질" 미검사 → 형식만 채우기 게이밍.
- **I10**: 한 줄 verdict에 미검증·⚪ 누락 → 불변식1 표현계층서 뚫림.

## 수정안 (순서: B3가 토대 — B4 집계·전수성·고위험합의 트리거가 의존)

### 1. B3 — 주장/표면 인벤토리 (§1에 산출물 추가)
- **추가**: §1 산출물로 "주장 인벤토리" — 검증 대상이 함의하는 주장 목록 + 각 주장이 닿는 **치명 표면**(auth·데이터·migration·결제·비가역 등) 표기.
- **소비처 2곳**: (a) §3.2 고위험 합의 트리거 = "각 치명 표면에 대해 B·C의 **명시** 답변"(무언급=미검토로 집계). (b) §4 전수성 = 각 주장이 {VERIFIED/BLOCK/의견/사용자결정/미검증} 중 **정확히 하나**에 들어갔는지 개수 대조.

### 2. B1 — 비치명 단독 판단형 경로
- **고침**: §3.2 첫 불릿 `한 곳만 지적` → `치명 단독`으로 교체(트리거 목록과 일치).
- **추가**: §3.3 행 — `판단형, 토론 미발동(비치명 단독): 자기 4필드(claim·evidence·artifact·falsifier) 유효 → ⚪ 살림+중요도('무토론' 표기) / 무효 → drop`.

### 3. B2 — 치명·미해결 충돌
- **추가**: §3.3 행 `치명 클래스 + 미해결(근거 불명확 포함) → 🔴 BLOCK(임시)`.
- **한정**: 기존 행6 → `비치명 판단형 미해결 → 🟡`.
- **지위 명시**: 불변식 = "테이블이 이 성질을 만족하는지 **검산하는 조건**"(경쟁 verdict 아님).

### 4. B4 — 최종 게이트 집계 (§4에 고정 규칙 추가)
한 줄 verdict 위에 "게이트 결정" 한 줄, 고정:
- `🔴(임시 포함) ≥1` → **불통과(FAIL)**
- `⚪[BLOCKER] ≥1` 또는 `🟡 ≥1` → **HOLD(사용자 승인 전 머지 금지)**
- 그 외 → **통과(PASS)** (미검증 영역 목록 첨부 조건)

### 5. I2 — 중요도 함수화 (순환 제거)
- stake를 **유한 순서 스케일**: `demo < internal < published < prod`. floor도 이 값.
- 3축→등급 **lookup**: 정확성 바꿈+비가역→BLOCKER / 정확성 바꿈+가역→중요 / 표면적→사소. 그 뒤 floor로 하한 상향.
- 행7 **한정**: "선언 stake 자체가 분쟁/애매(floor 다툼)일 때만 🟡".

### 6. I6 — 필드 "실질" 검사
- `artifact` = 해석 가능한 **file:line 또는 실행 가능 명령**만(모호어 무효).
- `falsifier` = "X 실행해 Y면 철회" **결정 가능** 진술. 3R 내 결정 불가 → 필드 누락 처리.

### 7. I10 — 한 줄 verdict에 전 버킷 노출
- 포맷: `BLOCK n / VERIFIED k(커버 주장 수) / 의견살림 j(최고 중요도) / 사용자결정 m / 미검증 u`

## 수정 밖 (심각도 하향, 다음 라운드)
